# Tipos de dados 

 ## Estruturados

Banco de dados relacionais, como por exemplo Sql Server, 
Sql do Azure, e Sql Data warehouse, a estrutura de dados 
é definida em tempo de design. Criado em forma de tabela.
Sistemas relacionais são lentos para reagir a alterações nos 
requisitos de dados, já que o banco precisa mudar sempre
que há alteração no requisito de dados.

## Não estruturados

Arquivos de audio, de imagem e binários são exemplos de 
dados não estruturados. São armazenados em sistemas 
relacionais, chamados não estruturais ou sistemas NoSql.
A estrutura de dados não é definida em tempo de design
e os dados normalmente são carregados em seu formato bruto.

*Sistemas relacionais também podem dar suporte a dados semi-
estruturados, como arquivos JSON.*

## Quatro tipos de banco de dados NoSql (Software livre):

**1° Repositório de chave-valor:** Armazena pares chave-valor
em uma estrutura de tabela.

**2° Banco de dados de documento:** Armazena documentos marcados
com metadados para auxiliar em pesquisas de documentos.

**3° Banco de dados de grafo:** Localiza relações entre os pontos
de dados usando estrutura composta por vértices e bordas.

**4° Banco de dados de coluna:** Armazena os dados com base em 
colunas em vez de linhas, já que as colunas podem ser
definidas em runtime da consulta, permite flexibilidade
nos dados que são retornados com bom desempenho.SS
